# Стратегия миграции на Laravel - Краткое руководство

> **Для владельца системы: что нужно знать о переходе на новую платформу**

---

## 🎯 Главное

Ваша существующая система из 186 PHP файлов (`HTDocs_legacy/s/`) будет **постепенно переписана** на современный фреймворк Laravel 12, сохраняя весь функционал и данные.

### Почему это важно

1. **Безопасность** - Старый код уязвим к атакам
2. **Скорость разработки** - Новые функции добавляются в 3-5 раз быстрее
3. **Современность** - Поддержка новых технологий
4. **Команда** - Легче найти разработчиков
### Что останется без изменений

- ✅ Все ваши данные в базах данных
- ✅ Весь функционал системы
- ✅ Многоязычность (русский/английский)
- ✅ Логика работы с публикациями, авторами, справочниками

### Что изменится

- 📱 Более современный и удобный интерфейс
- ⚡ Быстрее работает (оптимизация)
- 🔒 Безопаснее (защита встроена)
- 🛠️ Легче добавлять новые функции

## 📊 Ваша текущая система

### Что у вас есть сейчас

**Структура:**
```
HTDocs_legacy/
├── Apache/              # Веб-сервер (удаляется)
├── phpMyAdmin/          # Админка БД (удаляется)
├── s/                   # ВАШ КОД - 186 файлов
│   ├── UserEnter/      # Вход в систему
│   ├── MainTable/      # Публикации
│   ├── Catalogs/       # Справочники (авторы, издательства)
│   ├── DataBases/      # Управление БД
│   ├── Languages/      # Переводы
│   └── ...             # И ещё 27 модулей
└── Rosenbrock/         # Математический модуль (не трогаем)
```

**Базы данных (4 шт):**
1. `db_manager` - Система управления (пользователи, настройки, переводы)
2. `literature` - Основная БД (публикации, авторы, издательства)
3. `phys_math_contents` - Физика/математика
4. `trees` - Иерархические данные

**Объем данных (из SQL дампов - тестовые данные):**
- 11 публикаций (в production может быть больше)
- 19 авторов (в production может быть больше)
- ~476 переводов интерфейса × 3 языка (EN, RU, иврит) = 1,429 строк
- 25 алгоритмов парсинга данных

**Примечание:** SQL дампы содержат тестовые/демо данные. В реальной production БД объем может отличаться.

**Карта миграции кода:**
- Старая функция `mysqli_query()` → Новая `DB::query()` или `Model::find()`
- Старый `include 'file.php'` → Новый `use App\Models\Model`
- Старый `$_SESSION['key']` → Новый `session('key')`
- Старый `echo "<div>"` → Новый Blade `<div>{{ $var }}</div>`

Подробнее в разделе "Карта миграции" в DEVELOPER_GUIDE_RU.md

## ⚠️ Риски и как с ними справляемся

### Риск 1: Потеря данных

**Как избегаем:**
- Полное резервное копирование перед началом
- Тестирование на копии БД
- Валидация после миграции

**Вероятность:** Низкая ✅

### Риск 2: Ошибки в алгоритмах парсинга

**Как избегаем:**
- Документируем каждый алгоритм
- Сравниваем результаты со старой системой
- Автоматические тесты

**Вероятность:** Средняя ⚠️

### Риск 3: Превышение сроков

**Как избегаем:**
- Реалистичный план
- Еженедельный контроль
- Гибкость в приоритетах

**Вероятность:** Средняя ⚠️

### Риск 4: Пользователям не понравится

**Как избегаем:**
- Сохраняем привычный workflow
- Показываем демо на ранних этапах
- Учитываем обратную связь

**Вероятность:** Низкая ✅
