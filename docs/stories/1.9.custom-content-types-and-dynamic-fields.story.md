# Story 1.9: Custom Content Types and Dynamic Fields

## Status
In Progress (Tasks 1-6, 8, 10 Complete - Task 7 Pending)

## Story

**As an** administrator,
**I want** to define custom content types with unique metadata fields,
**so that** I can adapt the system to different types of publications beyond the default types.

## Acceptance Criteria

1. Admin interface for creating/editing content types (name, icon, description)
2. Admin can define custom fields per content type (field name, type, required, visibility)
3. Supported field types: Text, Number, Date, Dropdown, Multiselect, Boolean, Long Text
4. Custom fields appear in publication form when content type selected
5. Custom field values stored in polymorphic `custom_field_values` table
6. Custom fields searchable and filterable (if marked as public)
7. Custom fields display on publication detail page
8. Existing content types (Books, Magazines, Articles, Other) can be extended with custom fields

## Integration Verification

- IV1: Adding custom fields doesn't break existing publication CRUD operations
- IV2: Custom field values validate correctly based on field type
- IV3: Custom fields display properly in all three languages

### Database Schema Details

**custom_fields Table:**
```sql
CREATE TABLE custom_fields (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    content_type_id BIGINT UNSIGNED NOT NULL,
    field_name VARCHAR(255) NOT NULL,
    label_en VARCHAR(255) NOT NULL,
    label_ru VARCHAR(255),
    label_he VARCHAR(255),
    field_type ENUM('text', 'number', 'date', 'dropdown', 'multiselect', 'boolean', 'long_text') NOT NULL,
    field_config JSON,
    is_required BOOLEAN DEFAULT FALSE,
    visibility ENUM('public', 'admin_only', 'hidden') DEFAULT 'public',
    is_searchable BOOLEAN DEFAULT FALSE,
    is_filterable BOOLEAN DEFAULT FALSE,
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    deleted_at TIMESTAMP,
    FOREIGN KEY (content_type_id) REFERENCES content_types(id) ON DELETE CASCADE,
    INDEX idx_content_type_id (content_type_id),
    INDEX idx_sort_order (sort_order)
);
```

**custom_field_values Table:**
```sql
CREATE TABLE custom_field_values (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    custom_field_id BIGINT UNSIGNED NOT NULL,
    fieldable_type VARCHAR(255) NOT NULL,
    fieldable_id BIGINT UNSIGNED NOT NULL,
    value JSON,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    FOREIGN KEY (custom_field_id) REFERENCES custom_fields(id) ON DELETE CASCADE,
    INDEX idx_fieldable (fieldable_type, fieldable_id),
    UNIQUE KEY unique_field_value (custom_field_id, fieldable_type, fieldable_id)
);
```

### Search and Filter Integration

**Global Search Integration:**
[Source: app/Livewire/Search/GlobalSearch.php]
- Extend query to join `custom_field_values` table
- Include only searchable custom fields (is_searchable=true)
- Use JSON_SEARCH for value matching
- Example query:
```php
$query->whereHas('customFieldValues', function($q) use ($searchTerm) {
    $q->whereHas('customField', fn($cf) => $cf->where('is_searchable', true))
      ->whereRaw("JSON_SEARCH(value, 'one', ?) IS NOT NULL", ["%{$searchTerm}%"]);
});
```

**Filter Integration:**

```php
$query->whereHas('customFieldValues', function($q) use ($fieldId, $filterValue) {

});
```

### Validation Rules

**Custom Field Validation:**
- field_name: required, alpha_dash, unique per content_type_id
- label_en: required, max:255
- field_type: required, in:text,number,date,dropdown,multiselect,boolean,long_text
- is_required: boolean
- visibility: required, in:public,admin_only,hidden
- is_searchable: boolean (must be false if visibility=hidden)
- is_filterable: boolean (must be false if visibility=hidden)
- field_config: required_if field_type is dropdown or multiselect, must be valid JSON

**Custom Field Value Validation:**
- Depends on field_type:
  - text: string, max:255 (or max from field_config)
  - number: numeric, min/max from field_config
  - date: date_format:Y-m-d, between min_date and max_date from field_config
  - dropdown: in:options keys from field_config
  - multiselect: array, each value in:options keys from field_config
  - boolean: boolean
  - long_text: string, max:10000

### Performance Considerations

**Query Optimization:**
[Source: docs/architecture/coding-standards.md]
- Eager load custom fields: `ContentType::with('customFields')->get()`
- Eager load custom field values: `Publication::with('customFieldValues.customField')->get()`
- Use indexes on foreign keys and polymorphic columns
- Limit custom field queries to relevant content types

**Caching:**
- Cache content types with custom fields (rarely change)
- Cache filterable custom fields list
- Cache dropdown options for custom fields

**N+1 Query Prevention:**
```php
// ✅ GOOD
$publications = Publication::with([
    'contentType.customFields',
    'customFieldValues.customField'
])->get();

// ❌ BAD
$publications = Publication::all();
foreach ($publications as $pub) {
    $customFields = $pub->contentType->customFields; // N+1
}
```
