### Existing Database Schema

**Primary Keys**: All models use custom primary key names (not `id`):
- `publications.id_publication`
- `authors.id_author`
- `author_groups.id_author_group`
- `publishings.id_publishing`
- `magazines.id_magazine`
- `themes.id_theme`
- `theme_sets.id_theme_set`
- `parts.id_part`
- `part_sets.id_part_set`
- `issue_types.id_issue_type`
- `files` has composite key: `(id_publication, file_name)`

[Source: [architecture/4-data-models-and-schema-changes.md](../architecture/4-data-models-and-schema-changes.md#42-schema-integration-strategy)]

**Legacy Soft Delete**: The current system uses `_del_mark` tinyint column instead of Laravel's SoftDeletes. This story transitions to proper SoftDeletes trait while maintaining backward compatibility during migration.

**Lowercase Columns**: Existing pattern uses `*_low` columns for case-insensitive searches (e.g., `title_low`, `author_low`, `file_name_low`). This pattern should be maintained via model boot methods.

### Model Relationship Patterns

[Source: [architecture/8-coding-standards.md](../architecture/8-coding-standards.md#83-critical-integration-rules)]

**CRITICAL**: All Eloquent relationships MUST specify explicit foreign and local key parameters because this system uses non-standard primary keys:

```php
// ✅ CORRECT - Explicit keys required
public function publication(): BelongsTo
{
    return $this->belongsTo(Publication::class, 'id_publication', 'id_publication');
}

// ❌ WRONG - Will fail due to non-standard keys
public function publication(): BelongsTo
{
    return $this->belongsTo(Publication::class);
}
```

**Foreign Key Constraints**: When creating migrations with foreign keys:

```php
// ✅ CORRECT
$table->foreignId('publication_id')
    ->constrained('publications', 'id_publication');

// ❌ WRONG - Assumes 'id' column
$table->foreignId('publication_id')->constrained();
```

### Many-to-Many Pivot Table Requirements

[Source: [architecture/4-data-models-and-schema-changes.md](../architecture/4-data-models-and-schema-changes.md#41-new-data-models)]

The current system uses "set" tables (AuthorGroup, ThemeSet) which appear to store comma-separated values. This story creates proper many-to-many pivot tables:

**Author-Publication Pivot Table**:
- Table name: `author_publication`
- Columns: `id_author` (FK to authors.id_author), `id_publication` (FK to publications.id_publication), `order` (for author ordering), `timestamps`
- Indexes: Composite unique on `(id_author, id_publication)`, individual indexes on each FK

**Theme-Publication Pivot Table**:
- Table name: `publication_theme`
- Columns: `id_publication` (FK), `id_theme` (FK), `timestamps`
- Indexes: Composite unique on `(id_publication, id_theme)`, individual indexes on each FK

### File Storage and Metadata

[Source: [architecture/4-data-models-and-schema-changes.md](../architecture/4-data-models-and-schema-changes.md#42-schema-integration-strategy)]

New columns needed for files table:
- `mime_type` VARCHAR(100) - Store MIME type for validation
- `file_size_bytes` BIGINT - Store file size for display and quotas

New columns needed for publications table:
- `word_count` INT - Store calculated word count

**Testing Framework**: PHPUnit 11.5.3 with Laravel test helpers

**Testing Pattern for Relationships**:
```php
public function test_publication_has_many_authors()
{
    $publication = Publication::factory()->create();
    $authors = Author::factory()->count(3)->create();

    $publication->authors()->attach($authors->pluck('id_author'));

    $this->assertCount(3, $publication->authors);
    $this->assertInstanceOf(Author::class, $publication->authors->first());
}

public function test_author_belongs_to_many_publications()
{
    $author = Author::factory()->create();
    $publications = Publication::factory()->count(2)->create();

    $author->publications()->attach($publications->pluck('id_publication'));

    $this->assertCount(2, $author->publications);
}
```

**Testing Pattern for Soft Deletes**:
```php
public function test_publication_soft_deletes()
{
    $publication = Publication::factory()->create();

    $publication->delete();

    $this->assertSoftDeleted('publications', ['id_publication' => $publication->id_publication]);
    $this->assertNotNull($publication->fresh()->deleted_at);
}
```

**Regression Testing**: Ensure existing Livewire components (PublicationList, PublicationForm) still work by running their test suites after all changes.
