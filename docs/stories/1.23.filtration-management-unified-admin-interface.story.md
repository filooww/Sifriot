# Story 1.23: Filtration Management - Unified Admin Interface

## Status
Ready for Review

## Story

**As an** administrator,
**I want** a unified Filtration Management interface with tabs for all publication filter criteria (Content Types, Genres, Themes, Categories, Authors, Publishers),
**So that** I can efficiently manage all taxonomy and filter data in one place with consistent UX and multilingual support.

## Acceptance Criteria

### 1. Unified Filtration Management Interface

1. Create route `/admin/filtration` with Livewire component `FiltrationManagement`
2. Update navigation: Add "Filtration" link to admin menu (replace any existing "Content Types" link)
3. Interface displays horizontal tabs: "Content Types", "Genres", "Themes", "Categories", "Authors", "Publishers"
4. Tab state persists via URL query parameter (e.g., `/admin/filtration?tab=genres`)
5. Default tab is "Content Types"
6. All tabs require authentication and admin role
7. Tab navigation uses Livewire for smooth transitions without full page reload
8. Active tab highlighted with distinct styling (border-bottom + text color)
9. All tab labels translated in all three languages (EN/RU/HE)
10. Dark mode supported throughout with proper contrast

### 2. Content Types Tab

11. Content Types tab displays table with columns: Icon, Name (localized), Slug, Folder, System, Publication Count, Actions
12. Create button opens modal with form: name_en (required), name_ru, name_he, slug (required, unique), icon, folder_name, is_system (checkbox)
13. Edit action opens modal pre-populated with existing data
14. Delete action soft-deletes content type with confirmation (only if is_system=false AND no publications)
15. System content types (is_system=true) show lock icon, cannot be deleted
16. Slug auto-generates from name_en but remains editable
17. Flash messages display success/error for all operations

### 3. Genres Tab

18. Genres tab displays table with columns: Name (EN), Name (RU), Name (HE), Slug, Publication Count, Actions
19. Create button opens modal with form: name_en (required), name_ru, name_he, slug (required, unique)
20. Edit action opens modal pre-populated with existing data
21. Delete action removes genre with confirmation (only if no publications associated)
22. Slug auto-generates from name_en but remains editable
23. Flash messages display success/error for all operations

### 4. Themes Tab

24. Themes tab displays table with columns: Theme, Theme (lowercase), Publication Count, Actions
25. Create button opens modal with form: theme (required), theme_low (auto-populated, readonly)
26. Edit action opens modal pre-populated with existing data
27. Delete action removes theme with confirmation (only if no publications associated)
28. theme_low auto-populates as lowercase of theme field via Alpine.js
29. Uses legacy primary key `id_theme` throughout (queries, routes, bindings)
30. Flash messages display success/error for all operations

### 5. Categories Tab

31. Categories tab displays table with columns: Name (localized), Slug, Parent Category, Sort Order, Children Count, Publication Count, Actions
32. Create button opens modal with form: name_en (required), name_ru, name_he, slug (required, unique), parent_id (dropdown), sort_order (number)
33. Edit action opens modal pre-populated; parent dropdown excludes current category (prevent circular refs)
34. Delete action soft-deletes with confirmation (only if no children AND no publications)
35. Parent category dropdown shows "-" option for top-level
36. Parent column displays parent's localized name or "-" if top-level
37. Slug auto-generates from name_en but remains editable
38. Flash messages display success/error for all operations

### 6. Authors Tab

39. Authors tab displays table with columns: Author Name, Author (lowercase), Publication Count, Actions
40. Create button opens modal with form: author (required), author_low (auto-populated, readonly)
41. Edit action opens modal pre-populated with existing data
42. Delete action soft-deletes with confirmation (only if no publications associated)
43. author_low auto-populates as lowercase of author field via Alpine.js
44. Uses legacy primary key `id_author` throughout
45. Flash messages display success/error for all operations

### 7. Publishers Tab (NEW)

46. Create `publishers` migration with: id, name_en, name_ru, name_he, slug, website (nullable), timestamps, soft_deletes
47. Create `Publisher` model with publications() relationship via `publisher_publication` pivot table
48. Create `publisher_publication` pivot migration if not exists
49. Publishers tab displays table with columns: Name (localized), Slug, Website, Publication Count, Actions
50. Create button opens modal with form: name_en (required), name_ru, name_he, slug (required, unique), website (optional URL)
51. Edit action opens modal pre-populated with existing data
52. Delete action soft-deletes with confirmation (only if no publications associated)
53. Slug auto-generates from name_en but remains editable
54. Website field validates as URL if provided
55. Flash messages display success/error for all operations

### 8. Validation and Security

56. All tabs enforce admin-only access with `abort_unless(auth()->user()->isAdmin(), 403)`
57. Validation prevents duplicate slugs within same table
58. Deletion prevented if entity has associated publications
59. Category deletion prevented if entity has child categories
60. CSRF protection on all forms (Livewire default)
61. Input sanitization via Eloquent mass assignment protection

### 9. PublicationFilters Integration (Sidebar Filters)

62. PublicationFilters component updated to include Categories filter (multi-select with hierarchy display)
63. PublicationFilters component updated to include Publishers filter (multi-select)
64. Applied filters display shows selected categories and publishers as removable tags
65. Filter state persists in URL query parameters
66. Filters emit updated criteria via `filtersChanged` event

### 10. PublicCatalog (Root Page) Integration

67. PublicCatalog applies category filters: `whereHas('categories', ...)`
68. PublicCatalog applies publisher filters: `whereHas('publishers', ...)`
69. Publication cards optionally display categories and publishers

### 11. AdminDashboard Integration

70. AdminDashboard applies same filter criteria as PublicCatalog
71. Admin-specific filters (metadata status, etc.) continue working alongside new filters

### 12. MetadataReviewForm (Publication Edit) Integration

72. MetadataReviewForm includes Categories autocomplete multi-select
73. MetadataReviewForm includes Publishers autocomplete multi-select (using Publisher model, not just text)
74. Form saves category associations to `category_publication` pivot
75. Form saves publisher associations to `publisher_publication` pivot
76. Existing publication loads its categories and publishers for editing

### 13. Publication Detail Display Integration

77. PublicationDetail/Preview displays associated categories as tags
78. PublicationDetail/Preview displays associated publishers (with website link if available)

### 14. Integration Verification

- IV1: Existing ContentTypeManager functionality preserved (route redirect or removal)
- IV2: Publication forms can select all filter criteria (content type, genres, themes, categories, authors, publishers)
- IV3: All filter criteria display correctly in publication detail pages
- IV4: Search/filter functionality works with all criteria types
- IV5: URL filter state shareable and bookmarkable

### Tab URL State with Livewire

```php
use Livewire\Attributes\Url;

class FiltrationManagement extends Component
{
    #[Url(as: 'tab')]
    public string $activeTab = 'content-types';

    public function switchTab(string $tab): void
    {
        $this->activeTab = $tab;
    }

    public function updatedActiveTab(): void
    {
        match($this->activeTab) {
            'content-types' => $this->loadContentTypes(),
            'genres' => $this->loadGenres(),
            'themes' => $this->loadThemes(),
            'categories' => $this->loadCategories(),
            'authors' => $this->loadAuthors(),
            'publishers' => $this->loadPublishers(),
            default => $this->loadContentTypes(),
        };
    }
}
```

### Deletion Constraint Logic

```php
public function deleteCategory(int $id): void
{
    $category = Category::findOrFail($id);

    // Check children first
    if ($category->children()->count() > 0) {
        session()->flash('error', __('Cannot delete category with child categories'));
        return;
    }

    // Check publications
    if ($category->publications()->count() > 0) {
        session()->flash('error', __('Cannot delete category with associated publications'));
        return;
    }

    $category->delete(); // Soft delete
    session()->flash('message', __('Category deleted successfully'));
    $this->loadCategories();
}
```
