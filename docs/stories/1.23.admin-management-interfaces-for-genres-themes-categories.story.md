# Story 1.23: Admin Management Interfaces for Genres, Themes, and Categories

## Status
Approved

## Story

**As an** administrator,
**I want** a unified Content Management interface with tabs for Content Types, Genres, Themes, and Categories,
**so that** I can efficiently manage all taxonomy data in one place with consistent UX and multilingual support.

## Acceptance Criteria

### 1. Unified Content Management Interface

1. Rename existing route `/admin/content-types` to `/admin/content-management`
2. Update navigation link from "Content Types" to "Content Management"
3. Content Management interface displays horizontal tabs: "Content Types", "Genres", "Themes", "Categories"
4. Tab state persists via URL query parameter (e.g., `/admin/content-management?tab=genres`)
5. Default tab is "Content Types" (existing ContentTypeManager view)
6. All tabs require authentication and admin role
7. Tab navigation uses Livewire wire:navigate or Alpine.js for smooth transitions
8. Active tab highlighted with distinct styling
9. All tab labels translated in all three languages (EN/RU/HE)

### 2. Genre Manager Tab

11. Genre tab displays all records in a table with columns: Name (EN), Name (RU), Name (HE), Slug, Publication Count, Actions
12. Create button opens modal with form fields: name_en (required), name_ru, name_he, slug (required, unique)
13. Edit action opens modal pre-populated with existing genre data
14. Delete action removes genre with confirmation dialog (only if no publications associated)
15. Slug auto-generates from name_en (lowercase, hyphenated) but remains editable
16. Flash messages display success/error states for all operations
17. Table shows publication count using `withCount('publications')`
18. All text labels translated in all three languages (EN/RU/HE)
19. Dark mode supported with proper contrast

### 3. Theme Manager Tab

20. Theme tab displays all records in a table with columns: Theme, Theme (lowercase), Publication Count, Actions
21. Create button opens modal with form fields: theme (required), theme_low (auto-populated as lowercase)
22. Edit action opens modal pre-populated with existing theme data
23. Delete action removes theme with confirmation dialog (only if no publications associated)
24. theme_low field auto-populates as lowercase version of theme field (JavaScript/Alpine.js)
25. Uses legacy primary key `id_theme` throughout
26. Flash messages display success/error states for all operations
27. All text labels translated in all three languages (EN/RU/HE)
28. Dark mode supported with proper contrast

### 4. Category Manager Tab

29. Category tab displays all records in a table with columns: Name (EN), Name (RU), Name (HE), Slug, Parent Category, Sort Order, Children Count, Actions
30. Create button opens modal with form fields: name_en (required), name_ru, name_he, slug (required, unique), parent_id (nullable dropdown), sort_order (number)
31. Edit action opens modal pre-populated with existing category data
32. Delete action soft-deletes category with confirmation dialog (only if no children and no publications)
33. Parent category dropdown shows all categories except current category (prevent circular references)
34. Parent category column displays parent's localized name or "-" if top-level
35. Sort order field determines display order within same parent level
36. Children count shows number of subcategories using `withCount('children')`
37. Slug auto-generates from name_en but remains editable
38. Flash messages display success/error states for all operations
39. All text labels translated in all three languages (EN/RU/HE)
40. Dark mode supported with proper contrast
41. Soft-deleted categories not shown in list (no restore UI required for this story)

### 5. Validation and Security

42. Content Management interface enforces admin-only access with `abort_unless(auth()->user()->isAdmin(), 403)`
43. Validation rules prevent duplicate slugs within same table
44. Deletion prevented if entity has associated publications
45. Category deletion prevented if entity has child categories
46. CSRF protection on all forms (Livewire default)
47. Input sanitization via Eloquent mass assignment protection

### 6. Testing Requirements

48. Feature tests verify admin-only access (guest redirected, non-admin blocked with 403)
49. Feature tests verify tab navigation and URL parameter persistence
50. Feature tests verify CRUD operations (create, read, update, delete) for each tab
51. Feature tests verify validation rules (required fields, unique slugs, relationship constraints)
52. Feature tests verify deletion constraints (cannot delete if publications/children exist)
53. Unit tests verify model relationships work correctly
54. Browser tests (Dusk) verify tab switching, multilingual UI, and dark mode rendering

## Dev Notes

### Unified Tab-Based Architecture Overview
[Source: User requirements, app/Livewire/Admin/ContentTypeManager.php]

**CRITICAL ARCHITECTURAL CHANGE**: This story refactors the existing standalone ContentTypeManager into a unified Content Management interface with four tabs:

1. **Content Types** (existing ContentTypeManager functionality)
2. **Genres** (new)
3. **Themes** (new)
4. **Categories** (new)

**Implementation Approach:**
- Rename `ContentTypeManager` → `ContentManagement`
- Add tab navigation with `$activeTab` property (Livewire #[Url] attribute)
- Consolidate all CRUD operations into single component with tab-specific methods
- Share modal infrastructure across all tabs
- URL parameter persists tab state: `/admin/content-management?tab=genres`

**Key Benefits:**
- Consistent UX across all taxonomy management
- Single navigation entry point
- Shared modal, flash message, and validation infrastructure
- Easier to maintain and extend with future taxonomy types

### Reference Implementation Pattern
[Source: app/Livewire/Admin/ContentTypeManager.php, resources/views/livewire/admin/content-type-manager.blade.php]

The existing ContentTypeManager provides the foundational pattern to replicate for Genres, Themes, and Categories tabs:

**Component Structure:**
- Public properties for collection, modal state, editing entity, and form fields
- `mount()` method with admin authorization check
- `load{Entity}()` method using `withCount()` for relationships
- `create{Entity}()` method resets form and shows modal
- `edit{Entity}(int $id)` method loads entity data into form fields
- `save{Entity}()` method validates, creates/updates, flashes message, reloads data
- `delete{Entity}(int $id)` method with relationship checks and system protection
- `closeModal()` and `resetForm()` helper methods
- `rules()` method returns validation array with unique constraint handling

**View Structure:**
- Header with title and description
- Flash message display blocks (green for success, red for error)
- Create button triggers modal
- Data table with all relevant columns
- Actions column with Edit/Delete buttons
- Modal overlay with form (wire:click.stop prevents close on form click)
- Form fields with wire:model bindings
- Validation error display under each field
- Cancel/Submit buttons in modal footer

**Key Patterns to Replicate:**
1. Use `abort_unless(auth()->user()->isAdmin(), 403, 'Unauthorized')` for authorization
2. Use `session()->flash('message', '...')` for success messages
3. Use `session()->flash('error', '...')` for error messages
4. Use `withCount('relationshipName')` for displaying counts
5. Use `wire:confirm` attribute for delete confirmations
6. Use `dark:` Tailwind prefix for all color classes to support dark mode
7. Use `{{ __('Translation Key') }}` for all user-facing text
8. Follow kebab-case for Blade view filenames, PascalCase for PHP class names

### Tab Navigation Implementation Pattern
[Source: Livewire documentation, user requirements]

**Livewire URL State Management:**
```php
use Livewire\Attributes\Url;

class ContentManagement extends Component
{
    #[Url(as: 'tab')]
    public string $activeTab = 'content-types';

    public function switchTab(string $tab): void
    {
        $this->activeTab = $tab;
    }

    public function updatedActiveTab(): void
    {
        // Reload data when tab changes
        match($this->activeTab) {
            'content-types' => $this->loadContentTypes(),
            'genres' => $this->loadGenres(),
            'themes' => $this->loadThemes(),
            'categories' => $this->loadCategories(),
            default => $this->loadContentTypes(),
        };
    }
}
```

### Model-Specific Implementation Details

**Genre Model:**
[Source: app/Models/Genre.php]
- Table: `genres`
- Primary key: `id` (standard Laravel)
- Fillable: `name_en`, `name_ru`, `name_he`, `slug`
- Relationship: `publications()` BelongsToMany via `genre_publication` pivot
- No soft deletes
- No special fields beyond multilingual names and slug

**Theme Model:**
[Source: app/Models/Theme.php]
- Table: `themes`
- **CRITICAL**: Primary key: `id_theme` (legacy naming)
- Fillable: `theme`, `theme_low`
- Relationship: `publications()` BelongsToMany via `publication_theme` pivot
- No soft deletes
- **Special handling**: Must use `id_theme` in all queries, relationships, and route model binding
- theme_low should auto-populate as lowercase version of theme

**Category Model:**
[Source: app/Models/Category.php]
- Table: `categories`
- Primary key: `id` (standard Laravel)
- Fillable: `parent_id`, `name_en`, `name_ru`, `name_he`, `slug`, `sort_order`
- Relationships:
  - `parent()` BelongsTo Category
  - `children()` HasMany Category ordered by sort_order
  - `publications()` BelongsToMany via `category_publication` pivot
- **Uses SoftDeletes trait**
- Casts: `sort_order` => 'integer'
- Accessor: `getLocalizedNameAttribute()` returns name based on current locale

### Hierarchical Category Implementation
[Source: app/Models/Category.php, user requirements]

**Parent Selection Logic:**
```php
// In editCategory() method, exclude current category from parent options
public function editCategory(int $id): void
{
    $this->editingCategory = Category::findOrFail($id);
    // ... populate form fields ...

    // Load all categories EXCEPT current one to prevent circular reference
    $this->allCategories = Category::where('id', '!=', $id)->get();
}

// In createCategory() method, load all categories
public function createCategory(): void
{
    $this->resetForm();
    $this->allCategories = Category::all();
}
```

**Parent Display in Table:**
```blade
<td>{{ $category->parent?->localized_name ?? '-' }}</td>
```

**Children Count Display:**
```php
// In loadCategories()
$this->categories = Category::with('parent')
    ->withCount(['children', 'publications'])
    ->orderBy('sort_order')
    ->get();
```

**Deletion Validation:**
```php
public function deleteCategory(int $id): void
{
    $category = Category::findOrFail($id);

    if ($category->children()->count() > 0) {
        session()->flash('error', 'Cannot delete category with child categories.');
        return;
    }

    if ($category->publications()->count() > 0) {
        session()->flash('error', 'Cannot delete category with associated publications.');
        return;
    }

    $category->delete(); // Soft delete
    session()->flash('message', 'Category deleted successfully.');
    $this->loadCategories();
}
```

            ->set('name_en', 'Fiction')

            ->assertSet('showModal', false);

    }
}
