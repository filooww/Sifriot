# Story 1.26: Enhanced AI Filtration Metadata Extraction

## Status
Draft

## Story

**As an** administrator reviewing publication metadata,
**I want** the Gemini AI extraction to also determine content type, genres, themes, section, and issuer/publisher for each document,
**so that** all filtration attributes are populated automatically (preferably in Russian) and I can confirm or adjust them via the existing review UI rather than entering everything manually.

## Acceptance Criteria

### 1. Enhanced Gemini Prompt

1. The `GeminiMetadataExtractorService::buildPrompt()` method is updated to request additional fields in its JSON schema:
   - `content_type`: one of the predefined system content types (`"Книги"`, `"Журналы"`, `"Статьи"`, `"Другое"`) — AI must pick the best match
   - `genres`: array of genre strings (existing behaviour, already extracted)
   - `themes`: array of abstract content-level theme strings (replaces the current single `theme` field)
   - `section`: the single most appropriate navigation section from the predefined list (post-Story 1.25 sections: `"Избранное"`, `"Коллекции"`, `"Обзор"` and their children)
   - `issuer`: string or null — the issuing organization if distinct from publisher (e.g., for magazines, periodicals, institutional reports)
2. The prompt instructs Gemini to return values **in Russian** by default
3. The prompt includes the list of valid `content_type` values and valid `section` values so Gemini can map to them
4. The existing extracted fields (title, authors, publication_year, publisher, genres, description) remain unchanged in the prompt
5. Confidence scores are requested for all new fields (`content_type`, `themes`, `section`, `issuer`)

### 2. ExtractedMetadata DTO Enhancement

6. Add `content_type` field (single string with confidence) to `ExtractedMetadata`
7. Replace single `theme` field with `themes` array field (array of strings with confidence, same pattern as `genres`/`authors`)
8. Add `section` field (single string with confidence) to `ExtractedMetadata`
9. Add `issuer` field (single string with confidence) to `ExtractedMetadata`
10. Add getter methods: `getContentType()`, `getThemes()` (returns `string[]`), `getSection()`, `getIssuer()`
11. Add setter methods: `setContentType()`, `addTheme()`, `setSection()`, `setIssuer()`
12. Update `isEmpty()`, `toArray()`, `getConfidenceScores()`, `getHighestConfidenceFields()` to include new fields
13. Backward compatibility: `getTheme()` returns first theme from themes array (or null) so existing callers don't break

### 3. Response Parsing Update

14. `GeminiMetadataExtractorService::parseResponse()` handles the new fields from Gemini JSON response
15. `content_type` is validated against predefined content type names (Russian); if no match, stored as raw text for admin review
16. `themes` parsed as array (same pattern as genres parsing)
17. `section` stored as raw text suggestion; admin decides final mapping
18. `issuer` stored as raw text string

### 4. Content Type Resolution Helper

19. Create a helper method `resolveContentType(string $aiValue): ?int` in the service (or a dedicated resolver) that fuzzy-matches the AI-returned Russian content type name to the `content_types` table and returns the matching `id` (or null if no confident match)
20. The resolver checks `name_ru`, `name_en`, and `slug` columns for a match
21. Resolution is **suggested only** — admin confirms via existing UI; no automatic assignment

### 5. MetadataReviewDashboard Bulk AI Extraction Update

22. When bulk "Extract with AI" processes results, the new fields (content_type, themes, section, issuer) are stored in `FileMetadata.extracted_data` JSON alongside existing fields
23. The extraction method remains `'gemini_llm'`

### 6. MetadataReviewForm UI Integration

24. The `MetadataReviewForm` displays AI-extracted `content_type` as a suggestion badge next to the content type dropdown (admin can accept or choose differently)
25. `themes` array populates the theme input area (multiple theme tags)
26. `section` displays as a suggestion badge next to the section selector (admin confirms)
27. `issuer` populates a new issuer/publisher field or appends to publisher if same entity
28. All new fields show confidence badges (same color-coded system as existing fields)

### 7. Translation Keys

29. Add translation keys for new UI labels in `lang/en.json`, `lang/ru.json`, `lang/he.json`:
    - "AI suggested content type" / "ИИ предложил тип контента" / "סוג תוכן שהוצע ע\"י AI"
    - "AI suggested section" / "ИИ предложил раздел" / "מדור שהוצע ע\"י AI"
    - "AI suggested themes" / "ИИ предложил темы" / "נושאים שהוצעו ע\"י AI"
    - "Issuer" / "Издатель" / "מוציא לאור"
    - Additional labels as needed for the enhanced extraction display

### 8. Unit Tests

30. Update `GeminiMetadataExtractorServiceTest` with tests for new fields in prompt and response parsing
31. Update `ExtractedMetadata` tests for new getters/setters, backward compatibility of `getTheme()`
32. Test content type resolution helper with Russian, English, and fuzzy matching
33. Test that malformed new fields don't break extraction (graceful fallback)

## Dev Notes

### Dependency on Story 1.25
This story **requires Story 1.25 (Category to Section Refactoring) to be completed first**. The AI prompt references section names from the Section model (post-rename). If 1.25 is not deployed, the `sections` table won't exist and the section list in the prompt will be invalid.

### Current Gemini Prompt (Story 1.24)
[Source: app/Services/MetadataExtractors/GeminiMetadataExtractorService.php:92-104]

The current prompt requests: `title`, `authors`, `publication_year`, `publisher`, `genres`, `theme` (single), `description` with confidence scores. This story expands the prompt to add `content_type`, `themes` (array replacing single theme), `section`, `issuer`.

### Predefined Content Types
[Source: database/seeders/ContentTypeSeeder.php]

| slug | name_en | name_ru | name_he |
|------|---------|---------|---------|
| books | Books | Книги | ספרים |
| magazines | Magazines | Журналы | כתבי עת |
| articles | Articles | Статьи | מאמרים |
| other | Other | Другое | אחר |

These are system content types (`is_system = true`). The AI prompt must list these Russian names so Gemini picks the closest match.

### Predefined Sections (Post-Story 1.25)
[Source: Story 1.25 / SectionSeeder planned data]

| Parent (RU) | Children (RU) |
|-------------|---------------|
| Избранное | Выбор редакции, Новые поступления |
| Коллекции | По теме, По автору |
| Обзор | Недавние, Популярные |

The AI should suggest a section from this hierarchy. Since sections are navigational (not semantic), the AI's suggestion is inherently lower-confidence than content_type or genre suggestions. The prompt should instruct Gemini to pick the most appropriate section based on content nature.

### ExtractedMetadata DTO
[Source: app/Services/MetadataExtractors/ExtractedMetadata.php]

Current fields: `title`, `authors` (array), `publication_year`, `publisher`, `genres` (array), `theme` (single), `description`. The `theme → themes` migration must maintain backward compatibility since `getTheme()` may be called by existing code in `MetadataReviewForm` and `MetadataReviewDashboard`.

**Key pattern** for array fields (follow `authors`/`genres` pattern):
```php
private array $themes = [];
public function addTheme(string $value, float $confidence = 0.5): self
public function getThemes(): array // returns string[]
```

### Theme Model Considerations
[Source: app/Models/Theme.php]

The `Theme` model uses legacy primary key `id_theme` and has only `theme` + `theme_low` fields (no multilingual columns). AI-extracted theme strings are raw text — the admin uses existing autocomplete to match/create Theme records. No automatic Theme creation.

### Publisher vs Issuer
The existing `Publisher` model (`name_en`, `name_ru`, `name_he`, `slug`, `website`) covers traditional publishers. The new `issuer` field handles cases where the issuing entity differs from the publisher (e.g., a university issuing a research paper, a government agency issuing a report, a magazine's parent organization). In the UI, `issuer` is presented alongside `publisher` — admin decides whether they're the same entity or different.

### Russian-First Output
The prompt instructs Gemini to return genre names, theme names, and free-text values in Russian. This aligns with the library's primary user base. The admin can always adjust to other languages via the review form.
