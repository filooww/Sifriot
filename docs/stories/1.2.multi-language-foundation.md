# Story 1.2: Multi-Language Foundation with RTL Support

## Status
Ready for Review

## Story

**As a** user,
**I want** to switch the interface language between English, Russian, and Hebrew,
**so that** I can use the system in my preferred language.

## Acceptance Criteria

1. Language switcher component added to main navigation (flags or dropdown)
2. Language preference stored in session for guests, in user profile for authenticated users
3. All existing UI strings migrated to language files (`lang/en.json`, `lang/ru.json`, `lang/he.json`)
4. Hebrew language displays with proper RTL (right-to-left) layout using Tailwind RTL utilities
5. Date and number formats localize based on selected language
6. SetLocale middleware properly applied to all routes
7. Language files include all UI strings with complete translations for all three languages

## Integration Verification

- **IV1**: Switching language updates entire interface without breaking layouts
- **IV2**: Hebrew RTL mode properly mirrors layouts (navigation, forms, lists)
- **IV3**: Existing PublicationList and PublicationForm render correctly in all three languages

### RTL Support with Tailwind CSS
[Source: [tech-stack.md](../architecture/tech-stack.md), Tailwind CSS documentation]

**Tailwind RTL Configuration**:
Tailwind CSS 3.x has built-in RTL support. Use logical properties instead of directional:
- Use `ms-4` (margin-start) instead of `ml-4` (margin-left)
- Use `me-4` (margin-end) instead of `mr-4` (margin-right)
- Use `ps-4` (padding-start) instead of `pl-4` (padding-left)
- Use `pe-4` (padding-end) instead of `pr-4` (padding-right)
- Use `text-start` instead of `text-left`
- Use `text-end` instead of `text-right`

**HTML Direction Attribute**:
Add to `<html>` tag in both `app.blade.php` and `guest.blade.php`:
```html
<html lang="{{ str_replace('_', '-', app()->getLocale()) }}"
      dir="{{ app()->getLocale() === 'he' ? 'rtl' : 'ltr' }}">
```

### Hebrew Language Translation Notes

**RTL Text Direction**:
- Hebrew reads right-to-left
- Text should align to the right in Hebrew mode
- UI elements should mirror (e.g., navigation on right, content on left)

**Common Hebrew Translations** (for reference):
- "Publications" → "פרסומים"
- "Dashboard" → "לוח בקרה"
- "Search" → "חיפוש"
- "Log in" → "התחברות"
- "Register" → "הרשמה"
