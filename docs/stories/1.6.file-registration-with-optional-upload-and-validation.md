# Story 1.6: File Registration with Optional Upload and Validation

## Status
Ready for Review

## Story

**As an** administrator,
**I want** to register existing files on disk and optionally upload new files,
**so that** I can catalog the 1.1TB library and add new content when needed.

## Acceptance Criteria

1. Admin can browse server filesystem via folder tree interface (starting from configured base path)
2. Folder browser displays: folders, files (with format icons), file sizes, modification dates
3. Unregistered files highlighted distinctly from already-registered files
4. Admin can select individual file(s) and click "Register as Publication"
5. Registration form pre-populates with filename-based metadata suggestions
6. Admin can also upload new files via traditional upload form (single file)
7. Upload accepts PDF, EPUB, TXT, DOCX file types
8. File size validation (max 500MB per file)
9. File type validation (MIME type checking, not just extension)
10. Uploaded files stored in organized directory structure (`storage/app/content/{type}/`)
11. Only file path stored in database (not file contents)
12. Unique filename generation for uploads prevents collisions
13. Upload progress indicator for large files
14. Duplicate file path detection prevents registering same file twice
15. Registration creates draft publication record with "Pending" status

## Integration Verification

- **IV1**: File storage path configuration via `.env` works correctly for both registration and upload
- **IV2**: Registration handles filesystem errors gracefully (permission denied, path not found)
- **IV3**: Uploaded files accessible via storage filesystem abstraction (`Storage::get()`)
- **IV4**: Folder browser performs well with directories containing 1000+ files

### Data Models and Schema
[Source: [4-data-models-and-schema-changes.md](../architecture/4-data-models-and-schema-changes.md)]

**FileRegistrationLog Model** (NEW):
- Table: `file_registration_logs`
- Purpose: Audit trail for all file registrations (manual, upload, bulk scan)
- Columns: `id`, `publication_id` (nullable), `file_path` (unique), `registration_source`, `folder_scan_job_id` (nullable), `metadata_auto_extracted`, `status`, `error_message`, `registered_by`, `created_at`, `updated_at`
- Unique index on `file_path` prevents duplicate registrations
- Relationships: `belongsTo(Publication)`, `belongsTo(User, 'registered_by')`

**ContentType Model** (NEW):
- Table: `content_types`
- Purpose: Define content types (Books, Magazines, Articles, Other) with folder mapping
- Columns: `id`, `name_en`, `name_ru`, `name_he`, `slug` (unique), `icon`, `folder_name`, `is_system`, `created_at`, `updated_at`, `deleted_at`
- System types: Books, Magazines, Articles, Other (`is_system = true`)
- Folder mapping: `folder_name` determines storage subdirectory (e.g., 'books' → `storage/app/content/books/`)
- Relationships: `hasMany(Publication)`

**Publications Table** (extend existing):
- Add columns: `content_type_id` (foreign key to content_types), `status` (enum: published/hidden/pending, default 'pending')
- Add indexes: `status`, `content_type_id`
- New scopes: `scopePending()`, `scopePublished()`
- Relationships: `belongsTo(ContentType)`

**Files Table** (existing, from legacy schema):
- Stores file path references, NOT file contents
- Columns include: `id_file`, `file_path`, `file_format`, `file_size`
- Story 1.6 will add new columns in future migration: `mime_type`, `file_size_bytes`

**Storage Structure**:
```
storage/app/content/
├── books/
├── magazines/
├── articles/
└── other/
```
- Files organized by content type
- Unique filenames prevent collisions: `hash('sha256', $originalName . time()) . '.' . $extension`

```php
// ✅ GOOD - Use config and Storage facade
$basePath = config('library.storage.base_path');
$fullPath = Storage::disk('local')->path('content/' . $relativePath);

// ❌ BAD - Hardcoded paths

```

**Livewire Validation Pattern**:
```php
// ✅ GOOD - Inline validation with custom messages
$this->validate([
    'uploadedFile' => [
        'required',
        'file',
        'max:524288', // 500MB in KB
        'mimes:pdf,epub,txt,docx',
    ],
], [
    'uploadedFile.max' => __('File too large'),
    'uploadedFile.mimes' => __('Invalid file type'),
]);
```
